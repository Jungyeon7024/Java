TCP와 UDP
	네트워크 통신 프로토콜 중 두가지 주요한 프로토콜
	컴퓨터간 데이터 통신을 가능하게 함
	
	TCP(Transmission Control Protocol)
		연결 지향적 프로토콜
		데이터를 보내거나 받을 때 먼저 연결을 설정 데이터 전송이 완료되면 연결 종료
		데이터 손실을 최소화하기 위해 데이터 전송상태 확인
		재전송 및 순서 제어를 지원
		신뢰성이 높다.
		스트림 지향 프로토콜 데이터를 연속된 스트림으로 전송
		ex) 이메일 전송, 파일 다운로드 웹 브라우징 등
		
		
	UDP(User Datagram Protocol)
		비연결 지향적 프로토콜	
		연결 설정 없이 데이터를 전송하기 때문에 데이터 손실이 될 수 있고, 순서가 바뀔 수 있음
		데이터 패킷 단위로 전송되며, 각 패킷은 독립적이다
		신뢰성이 낮다.
		ex) 실시간 스트리밍, 온라인 게임 DNS 조회 등
		
	Socket
		클라이언트 측에서 사용
		클라이언트는 Socket 객체를 생성해 서버에 연결
		서버 IP주소와 포트번호 사용해서 특정 서버에 연결하는 역할
		클라이언트와 서버간의 데이터 통신을 가능하게 함
		getInputStream()  : 입력 스트림을 얻을 때 사용
		getOutputStream() : 출력 스트림을 얻을 때 사용
		connect()         : 클라이언트가 서버에 연결할 때 사용 주어진 서버 주소와 포트번호로 서버에 연결을 시
		close() 	      : Socket 객체를 닫을 때 사용
